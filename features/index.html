Features

Everything GitHub Store does

A discovery layer over GitHub releases, an install layer that silently delivers updates, and an honest backend that nothing about you is logged through.

Get GitHub Store Read the Wiki

Smart discovery

Curated index over the live GitHub Search API. Honest freshness with a passthrough fallback. Only repos that ship installable assets show up.

  • Home sections for Trending, Hot Release, and Most Popular projects with time-based filters.
  • Platform-aware topic scoring — Android users see APK-shipping repos first; desktop users see DMG/EXE/AppImage first.
  • Search filters for platform, language, and sort order, including Recently Updated (repo activity) and Recently Released (latest stable release date).
  • Search history saved locally and surfaced as suggestions.
  • Clipboard link detection — paste a GitHub URL anywhere on your device and the app offers quick navigation.
  • Hide seen repos — optional filter that removes repos you've already viewed from discovery feeds.
  • Add from starred — find APK-shipping repos in your GitHub stars and one-tap install them.

Release browser & install

Browse and install any release, not just the latest. Compatibility checks and APK introspection built in.

  • Release picker with the full release history per repository.
  • Single Install latest action plus an expandable list of all available releases and installers.
  • Manual install path with automatic compatibility checks.
  • APK Inspect (Android) — preview app label, signing fingerprint, version codes, min/target SDK, components, file metadata, and grouped permissions colour-coded by protection level — before you install.

Download mirror system

A multi-source race against the direct GitHub CDN. Verified end-to-end. Built for networks where github.com is throttled or blocked.

  • Multi-source race for the fastest available mirror.
  • End-to-end SHA-256 verification against GitHub's published asset digests.
  • Curated community mirror list, updatable from the backend without an app release.
  • Custom mirror URL for self-hosted gh-proxy-style instances.
  • Auto-suggest sheet after sustained slow downloads — opt in with one tap, dismissible, never nags.

Rich details

Everything a release page tells you — plus rendered READMEs, translation, signing-fingerprint pinning, and per-app variant filters.

  • App name, version, share action.
  • Stars, forks, open issues, license — sourced from the backend so you don't burn GitHub quota.
  • Rendered README ("About this app") with optional translation to your chosen language.
  • Release notes with full Markdown formatting per release.
  • Pre-release channel chip — toggle Include betas / Stable only per app, with a switch to stable rollback when a clean stable exists.
  • Merged "what's changed since v1.0" — concatenated release notes for every version skipped between updates.
  • Manual refresh — pull-to-refresh on Android, overflow-menu Refresh on every platform, Ctrl/Cmd+R on desktop. 30-second per-repo cooldown surfaces as a friendly retry-after toast.
  • Deep linking via githubstore://, github.com, or github-store.org.
  • Developer profile screen — explore a developer's repositories and activity.
  • Per-app variant pinning + asset filter regex — pin -arm64, -stable, -no-jvm, etc., and write a glob/regex filter so updates always pick the right artifact.

App management

Open, uninstall, downgrade, link existing apps, import from Obtainium, scan for GitHub-sourced apps already on the device.

  • Open, uninstall, downgrade installed apps directly from GitHub Store.
  • Library Imports (Android) — recognises GitHub-sourced apps already on the device (Obtainium, sideload, F-Droid). Three match strategies: manifest hint, signing fingerprint, backend lookup. High-confidence matches link silently; the rest land in a one-tap review wizard.
  • Obtainium import / export — bring your library over with one tap; export back to Obtainium any time. Round-trip preserved.
  • Link apps — connect any app already installed on your device to its GitHub repo so GitHub Store tracks updates for it.
  • Sectioned Apps screen — updates, pending installs, and installed apps grouped so you see what needs attention first.
  • Android: APK architecture matching (armv7/armv8), package monitoring, update tracking.
  • Android: Shizuku and Sui silent install — install and update apps without prompts (requires Shizuku or Sui running with ADB or root).
  • Android: Dhizuku silent install — Magisk-free silent installs on Xiaomi, OPPO, vivo, and Huawei stock ROMs via the Dhizuku Device Owner method.
  • Android: Signing-fingerprint verification for auto-updates — auto-update is blocked if the new APK's signing key doesn't match what the row was registered with. Prevents silent compromise on a hijacked release.
  • Android: Background update checking (3h / 6h / 12h / 24h) with notifications.
  • Android: Auto-update — silently installs available updates via Shizuku/Sui/Dhizuku when enabled.
  • Desktop: downloads installers to your Downloads folder and opens them with the default handler.

Collections

Personal libraries that sync locally — no account required for favourites and history.

  • Starred — browse your GitHub starred repositories from inside the app.
  • Favourites — save repos locally for quick access; works without sign-in.
  • Recently viewed — automatic per-device history of repos you've opened.

Cross-version communication

What's-new sheets and an in-app announcements feed that doesn't track you.

  • What's new sheet — one-shot release highlights on first launch after each update; permanent history under Profile → What's new.
  • Announcements feed — privacy notices, surveys, security advisories, and news in Profile. Anonymous backend feed; dismissal and read state stay on the device.

Authentication

OAuth that survives throttled networks. Personal Access Token fallback for users behind hard blocks.

  • GitHub OAuth via the device flow, routed through the backend by default so sign-in keeps working when github.com is throttled. Falls back to direct GitHub on infrastructure errors.
  • Personal Access Token sign-in for users whose network blocks the OAuth browser flow entirely. Validates locally, full feature parity with the OAuth path.

Tweaks

Theme, network, installer, translation, updates, storage, and feedback — all in one Settings screen.

  • Appearance — theme color picker (Dynamic, Ocean, Purple, Forest, Slate, Amber), light/dark/system, AMOLED black, system font toggle, scrollbar toggle (desktop).
  • Network — proxy configuration with HTTP/SOCKS support and optional auth; download-mirror picker with latency test.
  • Installation (Android) — choose default installer, Shizuku/Sui silent, or Dhizuku Device Owner, with real-time installer status. Installer attribution sub-setting lets you spoof the installer-package name silent installs claim — useful for apps that gate on install source.
  • Translation provider — switch between Google Translate (free, no key) and Youdao (custom app key + secret) for README and release-note translation.
  • Updates (Android) — update check interval, pre-release inclusion, auto-update toggle.
  • Storage — view and clear downloaded package cache.
  • Send feedback — bug reports and feature requests sent as email or pre-filled GitHub issues. The diagnostics card shows exactly what gets sent before you send it.

Localization

Available in 13 languages.

  • English, Arabic, Bengali, Chinese (Simplified), Spanish, French, Hindi, Italian, Japanese, Korean, Polish, Russian, Turkish.

Network & performance

An open-source backend, OS-level TLS, and a caching layer designed to never burn your GitHub API quota.

  • Dynamic proxy support (HTTP, SOCKS, System) for configurable network routing.
  • Enhanced caching system — faster loading and reduced API usage.
  • Anonymous backend at api.github-store.org for auth proxy, curated discovery, and the announcements feed. Open source under Apache 2.0 — self-hostable.
  • Desktop trusts OS-installed root certificates in addition to the JVM bundle. TLS-intercepting tools (Watt Toolkit, FastGithub, Fiddler, corporate MITM proxies) work without keytool gymnastics.
